EC2 Troubleshoot Notes - Honest Version

Goal:
Practice real-world EC2 troubleshooting. Document what worked, what didn’t, and what you learned.

---

1. EC2 Status
- Was the instance running on first try? [Yes/No]
- Did I have to start it manually? [Notes]

2. SSH Test
- Did SSH connect immediately? [Yes/No]
- If not, what caused the failure? (SG, KeyPair, subnet)
- How did I fix it?

3. Security Groups
- Inbound rules checked? [Notes]
- Outbound rules checked? [Notes]
- Any adjustments made?

4. Route Tables
- Public subnet connected to IGW? [Yes/No]
- Private subnet setup (if used)? [Notes]

5. System Logs
- Any startup errors? [Notes]
- Anything unexpected in logs?

6. Lessons Learned
- What broke first? [Notes]
- How did I debug it? [Notes]
- What would I do differently next time? [Notes]

> Be honest: showing real troubleshooting is more valuable than “everything worked.”
